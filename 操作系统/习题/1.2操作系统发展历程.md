# 03

在计算机组成原理（或操作系统基础）中，多道程序设计（Multiprogramming）允许多个程序同时驻留内存并交替执行，由此引发了一系列复杂的执行特性。以下逐一深入讲解。

---

### **顺序性（Sequentiality）**

顺序性是程序最基础的特性，指的是在**单道程序**环境下，程序的各条指令严格按照编写时规定的逻辑顺序依次执行，前一条指令执行完毕后才执行下一条，整个执行过程是一个连续、有序、可预测的序列。

这一特性是冯·诺依曼体系结构的核心体现——程序计数器（PC）逐步递增，控制器顺序取指、译码、执行。在此模式下，程序的行为完全确定，给定相同的输入，每次运行结果必然相同，调试和验证也相对容易。

然而，进入多道程序设计后，顺序性被打破。多个程序交替占用 CPU，单个程序的执行不再是连续的，这正是引出其他三个特性的根本原因。

---

### **间断性（Intermittency）**

间断性，也称为**异步性**，是多道程序设计环境下程序执行的典型特征。由于多个程序共享 CPU 和其他资源，某个程序在执行过程中可能因为等待 I/O 操作完成、等待某个事件发生或被高优先级程序抢占，而不得不**暂停执行**，让出处理器，之后在某个时刻再恢复运行。

这种"走走停停"的执行模式使得程序的执行时间变得不确定。一个程序从开始到结束，其实际占用的时钟时间（墙上时间）远大于它真正使用 CPU 的时间。间断性要求操作系统必须能够保存和恢复程序的执行现场（即**进程上下文切换**），确保程序在恢复执行时能从中断点继续，而不是从头开始。

---

### **共享性（Sharing）**

共享性指多道程序中的多个程序可以**同时或交替地使用**系统中的同一资源，包括内存、文件、I/O 设备、数据库等。共享是多道程序设计提高资源利用率的核心手段。

共享分为两种形式：
- **互斥共享**：某些资源（如打印机、某些临界区数据）在同一时刻只能被一个程序独占使用，其他程序必须等待，典型机制是互斥锁（Mutex）。
- **同时访问**：某些资源（如只读的代码段、共享内存区域）可以被多个程序同时读取而不造成冲突。

共享性带来了资源竞争问题，如果不加以管理，就会引发**死锁**、**数据不一致**等严重问题，这直接催生了制约性。

---

### **制约性（Constraint / Mutual Restriction）**

制约性是共享性和间断性共同作用的必然结果，指并发执行的多个程序之间存在**相互制约**的关系，这种制约体现在两个层面：

**直接制约**来源于程序间的**协作关系**。例如，生产者-消费者模型中，消费者必须等待生产者生产出数据后才能消费；管道通信中，读进程必须等待写进程先写入数据。这种制约是程序逻辑本身设计的，是有意为之的同步关系，通常通过**信号量（Semaphore）**或**条件变量**来实现。

**间接制约**来源于程序间的**资源竞争关系**。两个原本没有逻辑联系的程序，仅仅因为都需要使用同一台打印机或同一块共享内存，就必须相互等待。一个程序的推进速度受到另一个程序是否释放资源的制约。

制约性使得程序的执行结果可能依赖于各程序推进的相对速度，即产生**竞态条件（Race Condition）**，这是并发程序设计中最难处理的问题之一。

---

### **四者之间的内在联系**

这四个特性并非孤立存在，而是构成一个逻辑链条：**多道程序设计**打破了单程序的顺序性，引入了**间断性**；为了提高效率，多个程序必然**共享**资源；而共享与间断并存，就产生了程序间的**制约性**。理解这四者的关系，是理解操作系统进程管理、同步与互斥机制的重要基础。

# 09

#### **A. 加大时间片​**

时间片（Time Slice）是分时系统中 CPU 分配给每个用户进程的固定时间段。**加大时间片**意味着每个进程每次能占用 CPU 更长时间，但轮转一圈所需的总时间也随之增加。

根据响应时间公式 $T = n \times q$，时间片 q 增大，响应时间 T 也随之**增大**，系统对用户请求的反应会变慢。因此，加大时间片**不利于**改善响应时间，反而会使其恶化。正确做法是适当**缩小**时间片。

#### **B. 采用静态页式管理**

页式管理是内存管理的一种方式。**静态页式管理**指进程运行前将其所有页面一次性全部装入内存，运行期间不再换入换出。属于内存管理策略，与响应时间**无直接关联**。
#### **D. 代码可重入 ​**

**可重入代码（Reentrant Code）​** 又称纯代码，指可以被多个进程**同时共享同一份代码副本**而不会相互干扰的代码。其关键在于代码执行过程中不修改自身，所有需要修改的数据都存放在各进程私有的栈或数据区中。同样属于内存共享优化，与系统响应时间**无直接关联**。

#### **C 的核心逻辑**

- **优先级**的作用：重要的作业/进程被赋予高优先级，调度器会优先选择它们运行，使其能**尽快获得 CPU 响应**，缩短这类进程的等待时间。
- **非抢占式**的作用：一旦某个进程获得 CPU 开始执行，它不会被随意打断，可以**连续运行直到完成或主动等待**，避免了频繁上下文切换的开销，也保证了次要进程在获得 CPU 后不会迟迟无法完成。

两者组合的结果是：高优先级进程快速响应，低优先级进程获得 CPU 后也能顺利执行完毕，整体上**有利于改善系统的平均响应时间**
